# Your First Project: Pong! 🏓

:::warning[Some coding experience helps!]

This tutorial is written for people who are new to **FlixelGDX**, not
new to programming. If you have never written code before, we strongly
recommend spending an afternoon on the basics of **Java** or **Kotlin**
first — variables, classes, methods, `if` and `for`.
Anywhere is fine; some good free starting points (from W3Schools!):

- [W3Schools Java Tutorial](https://www.w3schools.com/java/default.asp)
- [W3Schools Kotlin Tutorial](https://www.w3schools.com/kotlin/index.php)

You don't need to be an expert — just comfortable reading a simple
class with a field, a constructor, and a method.

## Which one should I pick?

While both work flawlessly, don't use Java if you:

1. Are transitioning from **HaxeFlixel** (Kotlin's modern feature set closely mirrors Haxe).
2. Want a cleaner, more concise language that is easier for game development beginners.

Because of this, **Kotlin** is an obvious choice, <i>especially</i> if you're coming from the Haxe ecosystem.
When you combine FlixelGDX's simple, modular and backend-agnostic API with a simple, easy to use language, you
will immediately fall in love with the ecosystem and have a lot of fun making games!

:::

Hey there! In this tutorial, we're going to build the classic Pong
game in FlixelGDX. You know the one — two paddles on either side of the
screen, a tiny ball bouncing back and forth, and the first player to
miss the ball loses.

Pong is a perfect first project. It is small enough to finish in one
sitting, but along the way you will learn almost every important
FlixelGDX idea:

- 🎬 Creating and handling **sprites** inside a state.
- 🔄 **Switching states** and opening a **substate**.
- 💥 Handling **collision** logic and a bit of basic math.
- ✍️ Using the **text** object system to show the score.
- 🎮 Handling **input** from the keyboard.
- ⏱ Working with **elapsed / delta timing** so your game feels the same on every computer.
- 📝 Using the **logging API** to peek at what your game is doing.

:::tip[Pick your language]

Every code block on this page has tabs for **Java** and **Kotlin** —
the same two languages the [project generator](/getting-started)
supports. Pick whichever matches the project you generated, and stick
with it for the whole tutorial.

:::

## A quick mental model

Before we type a single line of code, let's set up a little picture in
your head.

Think of your game like a **TV show**.

- The **`FlixelGame`** is the **TV itself** — it stays on the whole time.
- A **`FlixelState`** is an **episode** — a self-contained scene like
  the main menu, the gameplay, or the credits.
- A **`FlixelSubState`** is a **commercial break** — it pauses what is
  on screen, plays a little extra scene (a pause menu, a "GAME OVER"
  message), then hands control back when it is done.
- Each **`FlixelSprite`** is an **actor** in the current episode — the
  ball, a paddle, a score number.
- **`Flixel`** (capital F) is the **remote control** — it lets us say
  *switch the channel* (`Flixel.switchState`), *listen to the
  microphone* (`Flixel.keys`), or *talk on the studio intercom*
  (`Flixel.info`).

Easy, right? Let's build the show.

## 2. Make some paddles

The first thing every Pong game needs is two **paddles** — one for the
player on the left, one for the enemy on the right. In FlixelGDX, the
quickest way to draw a colored rectangle on screen is the built-in
`makeGraphic` helper on `FlixelSprite`.

:::info[Quick analogy]

`makeGraphic(width, height, color)` is like saying:
*"Hey FlixelGDX, hand me a blank piece of paper this big, painted in
this color."* You can swap that for a real image later, but for now
colored rectangles are perfect.

:::

Open `PlayState` and replace the body with:

<Tabs groupId="lang">
<TabItem value="java" label="Java" default>

```java
package com.example.pong;

import com.badlogic.gdx.graphics.Color;
import org.flixelgdx.Flixel;
import org.flixelgdx.FlixelSprite;
import org.flixelgdx.FlixelState;

public class PlayState extends FlixelState {

  // Two paddles. We keep them as fields so we can use them every frame.
  private FlixelSprite player;
  private FlixelSprite enemy;

  @Override
  public void create() {
    super.create();
    Flixel.info("Pong is starting!");

    // Left paddle. This is the human player.
    player = new FlixelSprite(20, 200);
    player.makeGraphic(8, 64, Color.WHITE);
    add(player);

    // Right paddle. This is the enemy AI we'll code later!
    enemy = new FlixelSprite(612, 200);
    enemy.makeGraphic(8, 64, Color.WHITE);
    add(enemy);
  }
}
```

```kotlin
import org.flixelgdx.Flixel
import org.flixelgdx.FlixelSprite
import org.flixelgdx.FlixelState

    }
    add(player)

    // Right paddle. This is the enemy AI we'll code later!

    }
    add(enemy)
  }
}
```

</TabItem>

</Tabs>

**What just happened?**

- `create()` is the method FlixelGDX calls when your state first appears.
  Think of it as the **set-up scene** of the episode.
- `new FlixelSprite(x, y)` creates an actor at a position. The numbers
  are the **`x` / `y` of the sprite's bottom-left (not top-left like HaxeFlixel!) corner in world
  space** (as documented on
  [`FlixelObject`](/api/core/FlixelObject) in the API reference).
- `makeGraphic(8, 64, Color.WHITE)` paints a tall thin **white**
  rectangle. `Color` comes from libGDX
  (`com.badlogic.gdx.graphics.Color`); it ships handy constants like
  `Color.WHITE`, and you can build any color from a hex string with
  `Color.valueOf("ff2a3c")`.
- `add(player)` tells the state *"please include this actor when you
  update and draw the scene each frame"*.

Run the game (`./gradlew :lwjgl3:run`), and you should see two white paddles on either side of the screen.
Pong is starting to take shape!

## 3. Move the player paddle with the keyboard

Right now both paddles are statues. Let's teach the left one to listen
to **W** (up) and **S** (down). FlixelGDX's input lives on `Flixel.keys`,
and every state has an `update(float elapsed)` method that runs once per
frame.

:::info[Quick analogy]

`update(elapsed)` is the **director shouting "ACTION!"**
every frame. The `elapsed` parameter is **how many seconds passed since
the last shout** — usually a tiny number like `0.016`. We multiply our
speed by `elapsed` so the game feels the same on a slow laptop *and* a
fast desktop. That is what people mean by **delta time**.

:::

Add the `update` method below `create`:

<Tabs groupId="lang">
<TabItem value="java" label="Java" default>

```java
import org.flixelgdx.input.keyboard.FlixelKey;

// ...inside PlayState

private static final float PADDLE_SPEED = 220f;

@Override
public void update(float elapsed) {
  super.update(elapsed);

  if (Flixel.keys.pressed(FlixelKey.W)) player.setY(player.getY() - PADDLE_SPEED * elapsed);
  if (Flixel.keys.pressed(FlixelKey.S)) player.setY(player.getY() + PADDLE_SPEED * elapsed);

  // Don't let the paddle leave the screen.
  if (player.getY() < 0)        player.setY(0);
  if (player.getY() > 480 - 64) player.setY(480 - 64);
}
```

</TabItem>
<TabItem value="kotlin" label="Kotlin">


## 4. Add the ball, with collision

Now the fun part — a bouncing ball. We will give it a tiny `vx` / `vy`
velocity and bounce it off the top, bottom, and any paddle it touches.

:::tip

Velocity is just *"how many pixels per second the ball
wants to move"*. To move the ball, we add `velocity * elapsed` to its
position. Same delta-time trick as the paddle.

:::

Add a ball and two velocity fields, then use the built-in overlap check:

<Tabs groupId="lang">
<TabItem value="java" label="Java" default>

```java
private FlixelSprite ball;
private float ballVx = 200f;
private float ballVy = 140f;
```

Inside `create()`, after the paddles:

```java
ball = new FlixelSprite(316, 236);
ball.makeGraphic(8, 8, Color.valueOf("ff2a3c")); // FlixelGDX red
add(ball);
```

Inside `update(elapsed)`, after the paddle clamps:

```java
ball.setX(ball.getX() + ballVx * elapsed);
ball.setY(ball.getY() + ballVy * elapsed);

// Bounce off top/bottom.
if (ball.getY() < 0)       { ball.setY(0);       ballVy = -ballVy; }
if (ball.getY() > 480 - 8) { ball.setY(480 - 8); ballVy = -ballVy; }

// Bounce off paddles using the framework's AABB overlap check.
if (Flixel.overlap(ball, player) && ballVx < 0) ballVx = -ballVx;
if (Flixel.overlap(ball, enemy)  && ballVx > 0) ballVx = -ballVx;
```

`Flixel.overlap` runs the same AABB ("axis-aligned bounding box") check
you would write by hand — *"do these two rectangles touch?"* — but it
also quietly handles edge cases like sprites that have been `kill()`ed or
have `exists == false`, so your collision code stays one line.

> **Bonus:** when you want to handle a *whole group* of things, take a
> look at the top-level [`Flixel.overlap`](/api/) and
> [`Flixel.collide`](/api/) helpers. Those let you pass groups, get
> callbacks per pair, and even auto-separate the colliders.

Run the game. A red ball bounces between the paddles. 🟥

## 5. Give the enemy a brain

The enemy paddle should track the ball — but slowly enough that you can
beat it. We just slide it toward the ball's vertical center each frame:

<Tabs groupId="lang">
<TabItem value="java" label="Java" default>

```java
private static final float ENEMY_SPEED = 160f;

// inside update(elapsed):
float targetY = ball.getY() - 28; // 28 = half-paddle minus half-ball
if (enemy.getY() < targetY) enemy.setY(enemy.getY() + Math.min(ENEMY_SPEED * elapsed, targetY - enemy.getY()));
if (enemy.getY() > targetY) enemy.setY(enemy.getY() - Math.min(ENEMY_SPEED * elapsed, enemy.getY() - targetY));
if (enemy.getY() < 0)        enemy.setY(0);
if (enemy.getY() > 480 - 64) enemy.setY(480 - 64);
```

</TabItem>

## 6. Score! — using `FlixelText`

A Pong game with no score is just an animation. Let's add two big white
numbers at the top using **`FlixelText`**, the framework's text widget.

:::tip

Think of `FlixelText` as a **sprite with words on it**.

## 7. A pause substate

Finally, let's wire up a **pause screen** as a `FlixelSubState`. Press
**SPACE** to toggle pause; while paused, the gameplay freezes and a big
"PAUSED" label appears over the top.

> Remember the **commercial break** analogy? That is
> exactly what a substate is. The current state stays mounted *behind*
> the substate, so when the substate closes everything continues from
> where it stopped.

Create a new file `PauseSubState.java` (or `.kt`) next to
`PlayState`:

```java
import org.flixelgdx.FlixelSubState;
import org.flixelgdx.input.keyboard.FlixelKey;
import org.flixelgdx.text.FlixelText;

  @Override
  public void create() {
    super.create();
    FlixelText label = new FlixelText(220, 200, 200, "PAUSED");
    label.setTextSize(48);
    add(label);
  }

  @Override
  public void update(float elapsed) {
```

```java
package com.example.pong;

import com.badlogic.gdx.graphics.Color;
import org.flixelgdx.Flixel;
import org.flixelgdx.FlixelSprite;
import org.flixelgdx.FlixelState;
import org.flixelgdx.input.keyboard.FlixelKey;
import org.flixelgdx.text.FlixelText;

public class PlayState extends FlixelState {
  private static final float PADDLE_SPEED = 220f;
  private static final float ENEMY_SPEED  = 160f;

  private FlixelSprite player;
  private FlixelSprite enemy;
  private FlixelSprite ball;
  private float ballVx = 200f;
  private float ballVy = 140f;
  private int playerScore = 0;
  private int enemyScore  = 0;
  private FlixelText playerLabel;
  private FlixelText enemyLabel;

  @Override
  public void create() {
    super.create();
    Flixel.info("Pong is starting!");

    // Left paddle. This is the human player.
    player = new FlixelSprite(20, 200);
    player.makeGraphic(8, 64, Color.WHITE);
    add(player);

    // Right paddle. This is the enemy AI.
    enemy = new FlixelSprite(612, 200);
    enemy.makeGraphic(8, 64, Color.WHITE);
    add(enemy);

    ball = new FlixelSprite(316, 236);
    ball.makeGraphic(8, 8, Color.valueOf("ff2a3c"));
    add(ball);

    playerLabel = new FlixelText(150, 20, 200, "0");
    playerLabel.setTextSize(48);
    add(playerLabel);

    enemyLabel = new FlixelText(450, 20, 200, "0");
    enemyLabel.setTextSize(48);
    add(enemyLabel);
  }

  @Override
  public void update(float elapsed) {
    super.update(elapsed);

    if (Flixel.keys.justPressed(FlixelKey.SPACE)) {
      openSubState(new PauseSubState());
      return;
    }

    if (Flixel.keys.pressed(FlixelKey.W)) player.setY(player.getY() - PADDLE_SPEED * elapsed);
    if (Flixel.keys.pressed(FlixelKey.S)) player.setY(player.getY() + PADDLE_SPEED * elapsed);
    if (player.getY() < 0)        player.setY(0);
    if (player.getY() > 480 - 64) player.setY(480 - 64);

    ball.setX(ball.getX() + ballVx * elapsed);
    ball.setY(ball.getY() + ballVy * elapsed);
    if (ball.getY() < 0) {
      ball.setY(0);
      ballVy = -ballVy;
    }
    if (ball.getY() > 480 - 8) {
      ball.setY(480 - 8);
      ballVy = -ballVy;
    }

    if (Flixel.overlap(ball, player) && ballVx < 0) ballVx = -ballVx;
    if (Flixel.overlap(ball, enemy)  && ballVx > 0) ballVx = -ballVx;

    float targetY = ball.getY() - 28;
    if (enemy.getY() < targetY) enemy.setY(enemy.getY() + Math.min(ENEMY_SPEED * elapsed, targetY - enemy.getY()));
    if (enemy.getY() > targetY) enemy.setY(enemy.getY() - Math.min(ENEMY_SPEED * elapsed, enemy.getY() - targetY));
    if (enemy.getY() < 0)        enemy.setY(0);
    if (enemy.getY() > 480 - 64) enemy.setY(480 - 64);

    if (ball.getX() < 0) {
      enemyScore++;
      enemyLabel.setText(String.valueOf(enemyScore));
      resetBall(+1);
    }
    if (ball.getX() > 640 - 8) {
      playerScore++;
      playerLabel.setText(String.valueOf(playerScore));
      resetBall(-1);
    }
  }

      Flixel.info("Match over. Switching to GameOverState.");
```

```kotlin
import org.flixelgdx.Flixel

import org.flixelgdx.input.keyboard.FlixelKey
import org.flixelgdx.text.FlixelText

class PlayState : FlixelState() {
  companion object {
    private const val PADDLE_SPEED = 220f
    private const val ENEMY_SPEED  = 160f
  }

  private lateinit var player: FlixelSprite
  private lateinit var enemy: FlixelSprite
  private lateinit var ball: FlixelSprite

  private var ballVy = 140f
  private var playerScore = 0
  private var enemyScore  = 0
  private lateinit var playerLabel: FlixelText
  private lateinit var enemyLabel: FlixelText

    super.create()

    // Left paddle. This is the human player.
    player = FlixelSprite(20f, 200f).apply { makeGraphic(8, 64, Color.WHITE) }
    add(player)

    // Right paddle. This is the enemy AI.
    enemy = FlixelSprite(612f, 200f).apply { makeGraphic(8, 64, Color.WHITE) }
    add(enemy)

    ball = FlixelSprite(316f, 236f).apply { makeGraphic(8, 8, Color.valueOf("ff2a3c")) }
    add(ball)

    playerLabel = FlixelText(150f, 20f, 200f, "0").apply { setTextSize(48) }
    add(playerLabel)

    enemyLabel = FlixelText(450f, 20f, 200f, "0").apply { setTextSize(48) }
    add(enemyLabel)
  }

  override fun update(elapsed: Float) {
    super.update(elapsed)

    if (Flixel.keys.justPressed(FlixelKey.SPACE)) {
      openSubState(PauseSubState())
      return
    }

    if (Flixel.keys.pressed(FlixelKey.W)) player.y -= PADDLE_SPEED * elapsed
    if (Flixel.keys.pressed(FlixelKey.S)) player.y += PADDLE_SPEED * elapsed
    if (player.y < 0f)        player.y = 0f

    ball.x += ballVx * elapsed
    ball.y += ballVy * elapsed
    if (ball.y < 0f) {
      ball.y = 0f
      ballVy = -ballVy

    if (ball.y > 480f - 8f) {
      ball.y = 480f - 8f
      ballVy = -ballVy

    if (Flixel.overlap(ball, enemy)  && ballVx > 0) ballVx = -ballVx

    val targetY = ball.y - 28f
    if (enemy.y < targetY) enemy.y += minOf(ENEMY_SPEED * elapsed, targetY - enemy.y)
    if (enemy.y > targetY) enemy.y -= minOf(ENEMY_SPEED * elapsed, enemy.y - targetY)

    if (enemy.y > 480f - 64f) enemy.y = 480f - 64f

      enemyScore++

      resetBall(+1)

      playerScore++

      resetBall(-1)
```
