# flixelgdx-website

The website for the powerful Java game development framework
[**FlixelGDX**](https://github.com/flixelgdx/flixelgdx). Built with
[Docusaurus 3](https://docusaurus.io/) and styled to feel like a
GitHub-flavoured Javadoc page, with red and pitch-black as the
primary colors.

It hosts:

- A welcoming **home page** with feature highlights, animations, and
  per-section scroll fades.
- A **Getting Started** page with a client-side **project generator**
  that produces a runnable Gradle project (Java / Kotlin,
  IDE-specific configs, JitPack-wired FlixelGDX dependency, your choice
  of JDK vendor, bundled Gradle wrapper, expert-mode for power users).
- A **Your First Project** tutorial -- build Pong step by step, with a
  per-OS JDK install walk-through up top and synchronized
  Java / Kotlin code tabs throughout.
- An **API reference** generated from the framework's Java sources via
  [DocletMD](https://github.com/flixelgdx/DocletMD) and embedded as
  Docusaurus docs.

## Repository layout

```
.
├── gradlew                  <- bundled Gradle wrapper (no system Gradle needed)
├── gradlew.bat
├── gradle/wrapper/          <- gradle-wrapper.jar + properties
├── settings.gradle.kts      <- Gradle root settings (includes :docletmd)
├── build.gradle.kts         <- exposes generateAPI as a root-level alias task
├── docletmd/
│   └── build.gradle.kts     <- DocletMD runner that auto-discovers framework modules
├── scripts/
│   ├── build-api.sh         <- shallow-clones flixelgdx/flixelgdx + runs DocletMD
│   └── docletmd-init.gradle <- Gradle init script applied to the framework's own build
├── site/                    <- Docusaurus 3 project (npm)
│   ├── docs/                <- hand-written docs (getting-started, your-first-project)
│   ├── api/                 <- DocletMD output (generated by build-api.sh or generateAPI)
│   ├── src/components/      <- React components (project generator, JDK guide, ...)
│   ├── src/pages/index.tsx  <- home page
│   └── package.json
└── .github/workflows/
    └── deploy.yml            <- builds + deploys to GitHub Pages
```

## Prerequisites

- **Node.js 20+** -- the website uses Docusaurus 3.10, which requires
  Node >= 20. Run `nvm use` from the repo root to switch (an `.nvmrc`
  is included).
- **JDK 21+** -- needed by the bundled Gradle wrapper to run DocletMD and
  generate the API reference. We recommend [Eclipse Temurin](https://adoptium.net/)
  on contributor machines.

You do **not** need to install Gradle, DocletMD, or the FlixelGDX
framework -- `./gradlew` does that for you on demand.

## Local development

### Run the website

```bash
cd site
npm ci           # ~2 min on first run
npm run start    # http://localhost:3000/flixelgdx-website/
```

### Regenerate the API reference

The easiest way is the root Gradle task, which expects the framework
checked out at `../flixelgdx` (a sibling of this repo):

```bash
./gradlew generateAPI
```

Framework source elsewhere:

```bash
./gradlew generateAPI -PflixelgdxSrc=/absolute/path/to/flixelgdx
```

This will:

1. Locate the FlixelGDX source (sibling checkout, explicit path, or
   cached clone at `build/flixelgdx-src/`).
2. Apply `scripts/docletmd-init.gradle` to the framework's own Gradle
   build via the `-I` flag, so DocletMD is auto-wired to each module's
   source set and fully resolved compile classpath.
3. Copy the per-module output trees into `site/api/` (one subdirectory
   per module: `core`, `lwjgl3`, `teavm`).

The CI entry point `scripts/build-api.sh` does the same thing but first
performs a shallow clone of the framework:

```bash
./scripts/build-api.sh
```

Knobs (environment variables for `build-api.sh`):

```bash
FLIXELGDX_BRANCH=somebranch   ./scripts/build-api.sh
FLIXELGDX_REF=<commit-sha>    ./scripts/build-api.sh
FLIXELGDX_REPO=<git-url>      ./scripts/build-api.sh
```

If you have a local [DocletMD](https://github.com/flixelgdx/DocletMD)
checkout at `../DocletMD`, the task builds the plugin from source
automatically -- no `mavenLocal` publish needed.

### Production build

```bash
cd site
npm run build       # -> site/build/
npm run serve       # preview the static output locally
```

## Opening in IntelliJ IDEA

Just open the repo root. IntelliJ detects the `settings.gradle.kts`
file and offers to import the Gradle project -- accept. The
`generateAPI` task then shows up in the Gradle tool window and can be
run with a double-click.

The Node side stays a regular Docusaurus project; open the `site/`
folder as a Node module in WebStorm if you prefer dedicated JS tooling.

## Deployment

The site is deployed automatically to GitHub Pages from
[`.github/workflows/deploy.yml`](.github/workflows/deploy.yml). It
rebuilds on:

- Any push to this repo's `main` branch.
- A `repository_dispatch` event of type `framework-updated` (sent by
  the framework repo whenever `master` is updated).
- Manual `workflow_dispatch` invocations.
- A daily cron fallback.
